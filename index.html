O que é Client

< voltar

O termo client (cliente) refere-se ao dispositivo ou software que faz um pedido por um serviço ou recurso na internet. Pode ser:

  • Um navegador (Chrome, Firefox, etc.);
  • Um aplicativo de celular ou desktop;
  • Um computador, tablet ou smartphone.

Ou seja: sempre que você clica em um link, abre um site ou envia uma requisição — você está usando um client.

O client é responsável por interagir com o usuário — mostrar páginas, receber cliques, preencher formulários, exibir dados. Ele envia pedidos para um servidor, solicitando páginas, dados, arquivos ou funcionalidades.

Exemplos do dia a dia como client

  • Digitar um endereço de site no navegador,
  • Abrir um app como WhatsApp ou e-mail no celular,
  • Clicar em um link, preencher um formulário, pedir dados.

Tudo isso é client pedindo algo de server.


O que é Server

< voltar

O server (servidor) é o programa ou máquina que fornece serviços, dados ou recursos solicitados pelos clients. Ele “escuta” pedidos, processa requisições e devolve respostas.

Um servidor pode ser um computador poderoso, um conjunto de servidores, uma instância na nuvem, contêineres — enfim: uma infraestrutura preparada para atender múltiplos clientes simultaneamente.

O servidor cuida de processos críticos: dados, lógica de negócios, armazenamento, autenticação, geração de páginas dinâmicas, entre outros. Ele é “o backstage” que entrega o que o client consome.

Exemplos de servidor no cotidiano

  • O servidor de um site que entrega HTML/CSS/JS;
  • Servidores de e-mail que armazenam e enviam mensagens;
  • Servidores de banco de dados, APIs, armazenamento de arquivos.

Como funciona a interação Client ↔ Server

< voltar

A base do funcionamento entre client e server segue um padrão chamado request-response (requisição → resposta):

  1. O client envia uma requisição para o servidor (ex: “Me envie a página inicial do site”);
  2. O server recebe a requisição, processa (busca dados, aplica lógica, monta resposta);
  3. O server envia a resposta de volta (HTML, dados JSON, arquivos, etc.);
  4. O client recebe a resposta e a exibe para o usuário (navegador renderiza HTML, app mostra dados, etc.).

Esse modelo — conhecido como modelo cliente/servidor — é a base da internet moderna, web, serviços em nuvem, APIs, apps e muito mais.


Front-end e Back-end: responsabilidades de cada lado

< voltar

No contexto de desenvolvimento web, costuma-se dividir:

Front-end (lado do client): tudo o que o usuário vê — HTML, CSS, JavaScript, interface, interação. Ele consome dados do servidor e exibe de forma amigável.
Back-end (lado do server): lógica da aplicação, armazenamento de dados, regras de negócio, controle de acesso, processamento, etc.

Essa divisão permite manter o sistema organizado, seguro e eficiente — clientes leves e interativos, servidores potentes e confiáveis.


Vantagens e Limitações do Modelo Client-Server

< voltar

Vantagens:

  • Centralização de dados: tudo está no servidor — fácil de manter e atualizar.
  • Escalabilidade: servidores podem atender muitos clients ao mesmo tempo; basta aumentar recursos.
  • Independência de plataforma: client e server podem usar sistemas operacionais diferentes; só precisam comunicar por protocolo comum.
  • Segurança e controle: dados sensíveis ficam no servidor, com regras de acesso, backups e controle centralizado.

Limitações ou Desafios:

  • Ponto único de falha: se o servidor cair, todos os clients perdem acesso.
  • Sobrecarga: se muitos clients fizerem requisições simultâneas, o servidor pode ficar lento ou travar.
  • Dependência de rede: sem conexão, client não consegue comunicar com server — web fica indisponível.

Por que esse modelo é tão usado hoje?

< voltar

O modelo cliente/servidor virou padrão porque:

  • Permite separar claramente as responsabilidades entre interface e lógica/dados;
  • Facilita manutenção e escalabilidade;
  • Funciona em redes — local ou via internet — e suporta desde páginas simples até aplicações complexas, APIs, bancos de dados, nuvem, e muito mais;
  • Sustenta a web moderna, serviços em nuvem, aplicações distribuídas e o ecossistema de hoje.

HTML: o que é, para que serve e por que é tão importante na Web

< voltar

O HTML (HyperText Markup Language) é a linguagem base da Web. Toda página que você abre no navegador — seja um site simples ou um grande portal — utiliza HTML como estrutura fundamental. Criado para organizar e apresentar conteúdo na internet, ele define o que aparece na tela: textos, títulos, imagens, links, tabelas, formulários e muito mais.

HTML não é linguagem de programação! HTML é um arquivo de marcação.

Ao contrário de linguagens de programação, o HTML não possui lógica de decisão ou cálculos: ele é uma linguagem de marcação, composta por tags que estruturam os elementos de uma página. Cada tag cumpre um propósito específico, permitindo que navegadores interpretem e exibam corretamente o conteúdo.


Como o HTML funciona

< voltar

Um documento HTML é composto por elementos organizados entre , que indicam o significado e a função de cada parte da página. Por exemplo:

  • < h1> representa um título principal;
  • < p> representa um parágrafo;
  • < img> insere uma imagem;
  • < a> cria um link

Essas marcações formam a base estrutural da página, que depois pode ser estilizada com CSS e tornada interativa com JavaScript.


A evolução do HTML

< voltar

O HTML foi criado por Tim Berners-Lee no início da década de 1990 como parte de um sistema para tornar documentos científicos mais acessíveis. Desde então, a linguagem passou por diversas versões, até chegar ao HTML5, a versão moderna e padrão atual da Web.

O HTML5 trouxe recursos essenciais, incluindo:

  • suporte nativo para áudio e vídeo;
  • tags semânticas como < header>, < article>, < footer>;
  • melhor acessibilidade;
  • suporte mais amplo para APIs e aplicativos web.

Esse avanço tornou possível desenvolver sistemas complexos diretamente no navegador, como jogos, animações e aplicações web ricas.


Semântica: o grande diferencial do HTML moderno

< voltar

Um dos pilares da evolução do HTML é a semântica, isto é, o uso de tags que carregam significado. Em vez de apenas desenhar a estrutura, o HTML5 permite descrever o papel de cada elemento, ajudando:

  • a acessibilidade (leitores de tela interpretam melhor);
  • o SEO (motores de busca compreendem a página);
  • a organização do código.

Exemplos de tags semânticas modernas:

  • < header> – cabeçalho;
  • < nav> – menu de navegação
  • < section> – seção de conteúdo
  • < article> – conteúdo independente
  • < footer> – rodapé

Essa abordagem torna a Web mais estruturada, acessível e alinhada às boas práticas.


Explicação de algumas tags HTML

< voltar

Tags de Título – < h1> até < h6>

  • < h1> – Título principal (o mais importante)

  • < h2> – Subtítulo de nível 2

  • < h3> – Subtítulo de nível 3

  • < h4> – Subtítulo de nível 4

  • < h5> – Subtítulo de nível 5
  • < h6> – Subtítulo de nível 6 (menos importante)

Função: Semântica, hierarquia e organização de conteúdo, essencial para SEO.



< p> — Parágrafo

Usado para escrever textos em bloco, como frases e parágrafos.



< mark> — Texto marcado

Destaca um trecho como se estivesse marcado com marca-texto.



< small> — Texto pequeno

Exibe o texto em tamanho reduzido, útil para notas, direitos autorais, etc.



< i> — Itálico

Deixa o texto inclinado. Usado para palavras estrangeiras, termos técnicos, nomes científicos etc.



< u> — Sublinhado

Sublinha o texto. Use com moderação, pois pode confundir com links.



< b> — Negrito

Deixa o texto em destaque.



< strong> — Negrito semântico

Deixa o texto em destaque, mas com significado semântico de importância. É melhor que < b>.



< ol> — Listas Ordenadas

Cria uma lista numerada. Ex.: 1, 2, 3…



< ul> — Listas Não Ordenadas

Cria uma lista com marcadores (bolinhas).



< li> — Item de lista

Representa cada item dentro de < ol> ou < ul>.



< a> — Hyperlink

Cria links clicáveis para outras páginas, sites ou seções.

Atributos comuns:

  • href — URL do link
  • target="_blank" — abre em nova aba


< hr> — Linha horizontal

Cria uma linha para separar seções do conteúdo. Não precisa de fechamento.



< sub> — Subscrito (embaixo)

Texto menor e posicionado abaixo. Usado em fórmulas químicas, por exemplo: H< sub>2< /sub>O (H2O).



< sup> — Sobrescrito (em cima)

Texto menor e posicionado acima. Usado em potências: 10< sup>2< /sup> (102)



< blockquote> — Citação em bloco

Exibe uma citação longa, destacada e com recuo visual.
